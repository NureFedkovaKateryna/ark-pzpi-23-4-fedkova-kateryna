ДОДАТОК Б
VISION AND SCOPE DOCUMENT (КОНЦЕПЦІЯ ТА РАМКИ ПРОЄКТУ)
1 BUSINESS REQUIREMENTS (БІЗНЕС-ВИМОГИ)
1.1 Аналіз проблеми
У кавовому бізнесі значна частина процесів виконується вручну. Це може створювати суттєві проблеми, які впливають на якість обслуговування, ефективність роботи та фінансові показники бізнесу. Саме тому дуже важливо максимально автоматизувати ці процеси.
Перша проблема - це навчання персоналу. Нові робітники витрачають багато часу на запам’ятовування рецептів. Робітник може забути або переплутати які інгредієнти використовувати, в яких пропорціях та в якому порядку. Створення системи підказок допоможе зменшити кількість помилок.
Наступна проблема - це відсутність автоматизованого обліку інгредієнтів. Списання продуктів часто не фіксується точно. Через це власники не мають актуальної інформації про залишки. Виникають ситуації, коли популярні інгредієнти закінчуються в розпал робочого дня. Автоматичне списання інгредієнтів допоможе власникам контролювати залишки на складі після кожного замовлення, а не вручну змінювати інформацію в таблицях.
Наступна проблема - це відсутність інтеграції з обладнанням. Більшість кав’ярень не використовують IoT-інтеграцію. Кавомашини не взаємодіють з системою оформлення замовлень, тому процес приготування повністю залежить від людини. В результаті - це додаткове навантаження на персонал і довший час приготування замовлень.
Отже, створення системи, яка допомагає бариста готувати напої за стандартами, автоматично списує інгредієнти, контролює запаси в реальному часі, взаємодіє з кавовим обладнанням надасть багато можливостей для автоматизацїі кавового бізнесу.
1.2 Аналіз аналогів
Існує декілька систем автоматизації кофейного бізнесу, такі як: Toast, Square, Lightspeed.
Toast надає можливість генерувати звіти та має картки рецептур. Але ці картки орієнтовані тільки на ціноутворення, а не на покрокову підказку для бариста в момент приготування.
В Square є зручне управління меню, але немає підказок для покрокового приготування та інтеграції з IoT-модулем. 
В Lightspeed є функція інвентаризації, але немає підказок для покрокового приготування для бариста. 
На відміну від вже існуючих рішень, нова система буде надавати покрокові рецептурні підказки прямо в робочому інтерфейсі бариста та миттєво списувати інгредієнти після прийняття замовлення. Крім того, в новій системі буде підтримка IoT-модуля, який буде готувати основу напою, поки решту робить бариста за рецептом.
2 VISION OF THE SOLUTION (КОНЦЕПЦІЯ РІШЕННЯ)
2.1 Опис призначення системи
Система автоматизації кавового бізнесу призначена для ефективного управління процесами прийому та виконання замовлень, управлінням меню, а також для контролю запасів. Вона створена для використання власниками, адміністраторами та баристами, і надає інструменти, які дозволяють автоматизувати рутинні завдання, підвищити точність і швидкість обслуговування, а також оптимізувати бізнес-процеси кав’ярні.
Завдяки цій системі бариста отримують швидкий доступ до рецептів, що скорочує час на приготування і зменшує ризик помилок. Адміністратори та власники можуть легко контролювати запаси та ефективність персоналу, що дозволяє приймати більш обґрунтовані управлінські рішення.
2.2 Опис бізнес-логіки
Система автоматизації кавового бізнесу реалізує бізнес-процеси, пов’язані з прийомом і виконанням замовлень, управлінням запасами та контролем персоналу. Вона забезпечує взаємодію між користувачами та базою даних, автоматизуючи рутинні задачі та забезпечуючи точний облік ресурсів.
Дії, які можуть виконувати адміністратори та власники:
* реєстрація і керування користувачами та ролями;
* налаштування меню, цін та інгредієнтів;
* отримання повідомлень про низькі залишки на складі.
Дії, які можуть виконувати бариста:
* прийом та обробка замовлень;
* отримання автоматичних підказок по рецептах для кожного напою.
Бізнес-процес налаштування меню:
* власник або адміністратор додає, редагує та видаляє напої;
* кожен напій прив’язаний до інгредієнтів із зазначенням кількості для автоматичного списання та мінімальної кількості для повідомлень.
Бізнес-процес створення замовлень:
* бариста створює замовлення, обираючи продукти зі списку меню;
* система автоматично формує замовлення та підраховує загальну суму;
* статуси замовлення змінюються: в процесі, виконано;
* після завершення замовлення система списує інгредієнти.
Бізнес-процес керування складом:
* кожне приготування напою автоматично зменшує кількість інгредієнтів на складі;
* при надходженні нових поставок інгредієнти додаються в базу;
* система контролює рівень запасів та надсилає повідомлення, якщо кількість інгредієнтів менше мінімального числа.
Бізнес процес аналітики:
* отримання ефективності персоналу: кількість виконаних замовлень кожним баристою;
* витрати інгредієнтів: автоматичний підрахунок та історія списання.
3 SCOPE AND LIMITATIONS (РАМКИ ТА ОБМЕЖЕННЯ ПРОЄКТУ)
3.1 Опис функціональності серверної частини та IoT-клієнта
Сервер є центром обробки даних і логіки системи, який забезпечує взаємодію між користувачами, базою даних, аналітикою та IoT-пристроями. Завдання, які виконує сервер:
* управління користувачами (реєстрація та авторизація, призначення ролей і прав доступу, видача JWT-токенів для сесій);
* управління меню (CRUD-операції для напоїв, збереження рецептів та інгредієнтів для кожного напою);
* обробка замовлень (створення замовлення та списку товарів, розрахунок загальної вартості, зміна статусів замовлення);
* управління запасами (автоматичне списання інгредієнтів при приготуванні напою, додавання нових надходжень на склад, повідомлення про низький рівень запасів);
* аналітика та звіти (ефективність персоналу, витрати інгредієнтів);
* інтеграція з IoT (отримання даних від IoT-клієнтів про стан обладнання, відправка команд на IoT-пристрої для автоматизації процесу приготування напоїв).
IoT-клієнт - це проміжна ланка між сервером і кавовим обладнанням, який виконує команди сервера і передає дані назад. Завдання, які виконує IoT-клієнт:
* моніторинг стану обладнання (відстеження рівня води, молока, кавових зерен, виявлення помилок та збоїв в роботі);
* виконання команд сервера (автоматичний запуск приготування напою, відправка сигналів про завершення приготування);
* передача даних на сервер (поточний рівень інгредієнтів та стан обладнання, повідомлення про помилки або аварійні ситуації).
3.2 Опис функціональності веб-клієнта та мобільного клієнта
Веб-клієнт використовується власниками та адміністраторами для управління закладом, аналітики та моніторингу. Основні функції веб клієнта:
* управління користувачами (додавання користувачів, редагування даних, їх видалення, призначення ролей);
* управління меню (додавання та редагування напоїв, встановлення цін і активація/деактивація позицій меню);
* моніторинг замовлень (перегляд поточних замовлень у реальному часі, перегляд історії замовлень);
* аналітика та звіти (ефективність персоналу, витрати інгредієнтів);
* управління запасами (контроль залишків інгредієнтів, перегляд накладних приходу та списання продуктів);
* інтеграція з IoT (перегляд стану обладнання);
Веб-клієнт робить запити до серверного API, сервер обробляє ці запити, взаємодіє з базою даних та при необхідності надсилає команди IoT-клієнтам.
Мобільний клієнт орієнтований на бариста, щоб вони могли працювати з робочої станції або планшета. Основні функції мобільного клієнту:
* прийом і обробка замовлень (створення нових замовлень, відображення підказок по рецептах, миттєве оновлення статусу замовлень);
* автоматичне списання інгредієнтів після виконання замовлення;
* перегляд стану обладнання.
Мобільний клієнт отримує та надсилає дані через API серверу. Сервер обробляє замовлення, оновлює базу даних і надсилає команди на IoT-клієнти. 
4 BUSINESS CONTEXT (БІЗНЕС-КОНТЕКСТ)
4.3 Опис програмних засобів та інструментів
Для реалізації системи автоматизації кавового бізнесу буде використано сучасний набір технологій, який забезпечує гнучкість, масштабованість та зручність інтеграції з IoT-пристроями.
Основною мовою розробки серверної частини системи буде Python. У якості фреймворка REST API буде використовуватися Django та Django REST Framework. Ці інструменти забезпечують зручну структуру проєкту, швидку розробку REST API для взаємодії між мобільним, веб-інтерфейсом, IoT-модулем та серверною частиною. Для зберігання даних буде використовуватися PostgreSQL. Це реляційна СУБД, яка є надійною та має інтеграцію з Django.
Середовищем розробки буде Pycharm, який є інтегрованим середовищем розробки для Python. Для зручного керування версіями буде використовуватися Git. А для зберігання репозиторію та відстеження змін - GitHub. Це дозволить безпечно зберігати код, відслідковувати історію змін і розгортати систему.
Для інтеграції з IoT-модулем планується використати MQTT - протокол публікації та підписки, який підходить для IoT-рішень. Через MQTT сервер зможе надсилати команду кавомашині на приготування напою, отримувати підтвердження про статус приготування.
На боці IoT-пристрою буди використано MicroPython для простої обробки команд.
Для тестування REST API буде використовуватися Postman. Swagger UI буде використовуватися як автоматична документація для API.
